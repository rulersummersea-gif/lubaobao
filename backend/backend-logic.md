# 三主体后台业务逻辑梳理（企业/锅炉/检测包）

## 一、关系
1. 企业 1:N 锅炉
2. 企业 1:N 检测包
3. 锅炉 N:M 检测包（通过绑定表）

## 二、业务规则
- 锅炉创建必须归属企业
- 检测包创建必须归属企业
- 绑定时 boiler.enterprise_id == pack.enterprise_id
- 过期/作废/用尽检测包禁止绑定或巡检
- 企业停用后，禁止新增锅炉、激活/绑定检测包

## 三、后台管理菜单
1) 企业管理：企业档案、状态、统计看板
2) 锅炉管理：12字段档案、状态、绑定历史
3) 检测包管理：入库、激活、绑定、状态流转、库存预警
4) 绑定管理：绑定/解绑、冲突检测、历史追溯
5) 巡检与报告：任务状态、记录查询、月报聚合

## 四、关键流程
A. 企业入驻 -> B. 锅炉建档(12字段) -> C. 检测包入库/激活 -> D. 绑定锅炉 -> E. 发起巡检 -> F. 形成报告
